# Supabase Setup Instructions

This document provides the necessary SQL statements to set up the `tasks` table and its associated Row Level Security (RLS) policies in your Supabase project.

## 1. Create the `tasks` Table

Execute the following SQL statement in your Supabase SQL editor to create the `tasks` table:

```sql
CREATE TABLE tasks (
    id bigint generated by default as identity primary key,
    created_at timestamp with time zone default now() not null,
    title text not null,
    description text,
    status text default 'todo'::text not null,
    due_date date,
    assigned_to bigint references developers(id),
    project_id bigint references projects(id) on delete cascade
);
```

## 2. Enable Row Level Security (RLS)

First, enable RLS on the new `tasks` table:

```sql
ALTER TABLE tasks ENABLE ROW LEVEL SECURITY;
```

## 3. Create RLS Policies

These policies ensure that users can only access and manage tasks they are authorized to.

### a. Policy for SELECT (Read) Access

This policy allows users to see tasks that are assigned to them, or tasks associated with projects they are assigned to.

```sql
CREATE POLICY "Allow individual and project-based read access"
ON tasks
FOR SELECT
USING (
  auth.uid() IN (
    SELECT user_id FROM developers WHERE id = tasks.assigned_to
  )
  OR
  auth.uid() IN (
    SELECT user_id FROM developers
    JOIN projects ON developers.id = projects.assigned_to
    WHERE projects.id = tasks.project_id
  )
);
```

### b. Policy for INSERT (Create) Access

This policy allows any authenticated user to create new tasks.

```sql
CREATE POLICY "Allow authenticated users to create tasks"
ON tasks
FOR INSERT
WITH CHECK (auth.role() = 'authenticated');
```

### c. Policy for UPDATE (Modify) Access

This policy allows users to update tasks that are assigned to them.

```sql
CREATE POLICY "Allow users to update their own tasks"
ON tasks
FOR UPDATE
USING (
  auth.uid() IN (
    SELECT user_id FROM developers WHERE id = tasks.assigned_to
  )
);
```

### d. Policy for DELETE (Remove) Access

This policy allows users to delete tasks that are assigned to them.

```sql
CREATE POLICY "Allow users to delete their own tasks"
ON tasks
FOR DELETE
USING (
  auth.uid() IN (
    SELECT user_id FROM developers WHERE id = tasks.assigned_to
  )
);
```

By following these instructions, you will have a fully functional `tasks` table with appropriate security policies for your application.
